集成开发环境IDE

什么是IDE

是用于提供程序开发环境的应用程序，一般包括代码编辑器、编译器、调试器和图形用户界面等工具。集成了代码编写功能、分析功能、编译功能、调试功能等一体化的开发软件服务

  • IntelliJ IDEA：IDEA 全称 IntelliJ IDEA，业界被公认为最好的java开发工具之一，由JetBrains公司开发，旗舰版本不仅支持Java，还支持HTML，CSS，PHP，MySQL，Python等、收费软件
  • Eclipse：一个开放源代码的、基于Java的可扩展开发平台，最初是由IBM公司开发，2001年11月贡献给开源社区。免费软件
  • NetBeans：Java官方IDE。免费软件

    IDEA对比Eclipse优点：

  • 自动检索代码，并给出警告，eclipse需要手动启动代码分析
  • 自动提示
  • 智能调试
  • 跨语言操作更胜一筹

    IDEA对比Eclipse缺点：

  • 更占用硬件
  • 旗舰版收费，社区免费版功能不够用

IDEA下载旗舰版，学信网认证

IDEA基本配置

  • Appearance：显示设置，修改字体(UI里面的字，而不是代码里的字)，背景颜色等。
  • **Editor->General->Code Completion:**代码补全设置，默认就是大小写都提示补全。
  • Editor->General->Auto Import：自动导包，Ask下面两个勾上后，就不用手动导包了。
  • 新建项目，在Project SDK里选择JDK1.8
  • 在JAVAEE里选择web Application，然后一直下一步就行

遇到问题：

新建项目时，没有javaee和web Application

解决方法：

CTRL+alt+shift+/，选择registry，找到javaee.legacy.project.wizard

打上勾，再次创建就有javaee了，可以在里面选择web Application

IDEA创建多module的project

在IDEA中，project是最顶级的结构单元，然后就是module，一个project可以有多个module。目前，主流大型项目结构基本都是多module结构，这类项目一般是按功能划分的。比如：user-core-module，user-controller-module。模块之间彼此可以相互依赖，从命名可以看出，他妈都是处于一个项目中的模块。
一般情况下，IDEA是默认单module的。

如何创建多module的project

  • 新建项目，到命名那一步时，点击下面的More Setting，给原本和项目名相同的module name换成不同的名字。
    其他步骤与之前相同。
  • 点击File->new->module,勾选Web Application，命名一个新名字，下面的路径也要改名字

    最后点finish，就可以创建多个module

Java Web服务器的介绍

  • Web服务器：是指能够为发出请求的浏览器提供文档的程序。服务器是一种被动程序，只有浏览器发出请求的时候才会响应。

  • URL：统一资源定位符https://item.jd.com/10123735169.html

     - **协议：**http、https等
    • 域名或者IP地址：item.jd.com(域名) 116.211.94.1(JD的IP地址)
    • 端口号：80(常用端口)，程序必须使用端口号才能让另一台计算机访问（一般网址会省略）
    • 资源位置：用于描述web资源在服务器的位置，10123735169.html

Web服务器可以将一台计算机上的资源传输给我们

主流的Web服务器

  • WebLogic：Bea公司开发的一个收费的大型服务器软件，后被Oracle收购，支持Java EE所有规范。
  • Tomcat：支持全服JSP以及Servlet规范。学习java web一般使用tomcat
  • JBoss：一个开源的支持Java EE规范的应用服务器但是其核心服务不包含Servlet和JSP，所以一般情况下，它会与Tomcat等服务器绑定使用

#Tomcat的下载和使用
官网下载解压
bin—->startup.bat 双击打开，不要关闭，打开浏览器输入localhost:8080
看见页面就说明服务开启成功！
双击shutdown.bat 关闭服务

目录结构

  • bin：脚本目录，启动和关闭服务
  • conf：配置目录，web项目默认配置文件(server.xml,web.xml)
  • lib：Tomcat和web项目所依赖的jar文件
  • logs：日志文件
  • webapps：用于存放web项目目录
  • work：tomcat处理jsp页面的工作目录
  • temp：临时文件（可以删除不会影响app运行）

手动部署到Tomcat测试

在webapps里建立一个文件夹test，在里面建立a.txt
启动服务，在浏览器输入localhost:8080/test/a.txt
可以看到txt内容
同样也可以放网页文件等
