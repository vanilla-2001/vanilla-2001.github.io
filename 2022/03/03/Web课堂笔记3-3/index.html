Web课堂笔记3.3

发表于2022-03-03|更新于2022-05-25
|阅读量:

java se：单机
java ee：web开发
version control：版本控制
java enterprise：

  • project template：
  • rest service：资源模式，提供数据
  • Maven：架构，提供依赖插件
  • JUint：自动化测试框架
    Command Line App：命令行，web项目不需要
    工程名称不许起中文，也不能带空格，大小写无所谓
    src：放所有class

添加框架：

  • 在工程上右键 add framework support，添加框架支持
    web开发需要web application框架
    create web.xml添加配置文件
  • web目录：
  • 所有网络资源
  • web-inf：这个文件夹下的资源不能被用户直接访问

配置服务器：

右上角edit configuration
application server里选tomcat根目录，选择local，Name起名字
open browser：选择启动浏览器（选一个能用的浏览器），浏览器打开后会访问下面的网址，可以修改端口

部署（解决底下的warn）

点击 deployment –> + –> artitact

application context工程部署名称，可以修改的简单点，方便浏览器访问，还可以把整个工程名删掉，部署到根项目。
如果写了工程名，默认访问url后面会有工程名路径

Debug模式启动，可以通关断点方式追踪代码（run边上）
简单项目可以用run
默认访问的就是index.jsp里面的内容

配置：

  • file -> project structure ->project：
    comiler output:启动tomcat的虚拟目录
  • modules：
    可以新建文件夹配置成resource文件夹
  • libraries：
    添加tomcat文件夹找jsp和servlet，按住ctrl多选
  • artifacts：
    配置所有web（spring）框架
    可以添加删掉部署器

新建页面

在web下新建：名字.jsp

重启

redeploy:重新部署工程（内容少多选这个）
restar server：重新部署服务器（时间最久）
update classes and resources：更新文件

检查问题

在out里面检查最终部署的服务器，out中未更新，网页就访问不到，而不是看源码的
web-inf中的不可访问
